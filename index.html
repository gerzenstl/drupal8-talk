Drupal 8

“Es un sistema gestor de contenidos (CMS) open source que es desarrollado y mantenido por una comunidad de mas de 35.000 devs alrededor del mundo que han contribuido en el proyecto.”
Drupal 8

  • Creado por Dries Buytaert, en el 2000.

  • Open source CMF/CMS

  • Licencia GNU GPL

  • Altamente customizable si es requerido.

  • Enorme soporte de la comunidad.

  • Buena documentación y muchos recursos desde donde aprender.

  • Lanzamiento de Drupal 8: Noviempre 2015

Quien usa Drupal?
Drupal 8

Mejoras para usuarios finales.

  • Iniciativa SPARK.

  • WYSIWYG en el core (CKEditor).

  • Edición "in-place" de contenido.

  • Mobile First.

  • Responsive themes, image,...

  • Backend Mobile friendly.

Drupal 8

Mejoras para site builders.

  • Nuevo sistema de bloques.

  • Views, Entity reference y muchos otros módulos de la comunidad ahora en el Core.

  • Nuevos tipos de entidades.

  • Nuevos tipos de campos (datetime, link, phone, comments,..).

  • Forms incluyen tipos de display y mas mejoras.

  • Modulo Tour.

  • Multiples mejoras en Multilenguaje++.

Drupal 8

Mejoras para diseñadores y front-end devs

  • Nuevos elementos HTML5 en los fields.

  • Nuevas librerias integradas: Backbone.js, Modernizr, Twig, normalize.css

  • jQuery 2.0 (Desacoplado)

  • No mas soporte para IE 6 & IE 7 (y prácticamente IE 8).

Drupal 8

Mejoras para Drupal devs

  • Configuration Management (CMI).

  • Modulo de migración en Core (Migration path).

  • RESTFul Web services (en Core).

  • Nuevas APIs.

  • Performance++. Nuevas caches y Big Pipe (8.1).

  • Symfony components: HttpFoundation, HttpKernel, DependencyInjection, EventDispatcher, Routing, Yaml

Drupal 8

Mas Mejoras

  • OOP Avanzado, MVC, ...

  • "Proudly Found Elsewhere"

    • Codigo OOP moderno (clases, herencia, interfaces), MVC,

    • Ultimos standards en PHP (PSR-0, namespaces, traits).

    • Buenas librerias de 3ros: Composer, PHPunit, Guzzle, Zend Feed, ...

Drupal 8

Debo comenzar con Drupal 8?

  • Depende!

  • Complejidad/tamaño del proyecto.

  • Que modulos contrib va a necesitar?

  • Experiencia del desarrollador o equipo.

  • Site nuevo o existente (existente == migración)

  • Autores/contribuyentes de un modulo de la comunidad.

Breve demo de una instalación

  1. 1) Comenzar con drupal -> https://www.drupal.org/start

  2. 2) Verificar los requerimientos mínimos -> https://www.drupal.org/requirements

  3. 3) Descargar .tar.gz o .zip

  4. 4) Descomprimir archivo (dentro de la carpeta donde el web server tenga como root o mejor aún un VirtualHost)

  5. 5) Crear una base de datos con correspondiente usuario a la misma.

  6. 6) Acceder al sitio Drupal.

  7. 7) Completar los pasos de instalación.

  8. 8) A comenzar a armar ese sitio!
